制作自己的加密狗 - 一步一步来

按照说明依次操作。最后，将完成的加密狗插入路由器的 USB 端口，就可以在后台静静地阻止垃圾电话了。

Status: Beta. 这些说明是为经过测试的单一硬件类型量身定制的--购买步骤 1 中的电路板，那么接下来的每个步骤都将适用。其他电路板通常也能正常工作，但需要不同的驱动程序和你自己的侦查工作（请参阅下面的链接）。

此浏览器无法闪存加密狗。网络闪存仅适用于电脑上的 Google Chrome 浏览器或 Microsoft Edge 浏览器（不适用于智能手机）。使用 Chrome 或 Edge 打开此页面，然后再试一次。

开始之前

请简要检查您是否满足这四点要求，否则即使是完成的加密狗对您也毫无用处：

  • SIP 注册器 - 路由器（如 Fritz！Box）或直接与电话供应商连接，前提是允许外部 SIP 客户端（"可注册 IP 电话"）。
  • 2.4 GHz WLAN USB 端口范围内。
  • 5 V USB 电源 - 最好直接安装在路由器上，也可选择任何 USB 电源装置。
  • 免费 PhoneBlock 帐户用于 API 令牌。

步骤 1 - 订购加密狗

订购这款 ESP32 板。它非常小巧，可以作为 U 盘直接插入路由器，具有相应的 USB 串行芯片（CH340），PhoneBlock 加密狗就是用它开发和测试的。

从中国发货 - 约需 3-4 周。目前价格约为 9 欧元（含运费）。

在阿里巴巴订购

您想使用不同的电路板 其他 ESP32 DevKits 的工作方式完全相同，但如果设计不能直接插入路由器，则可能需要不同的 USB 串行驱动程序（例如用 CP2102 代替 CH340）和 USB 延长线。详细信息和建议：repo 中的 HARDWARE.md。

第 2 步 - 安装 USB 驱动程序（仅限 Windows）

加密狗使用 CH340 USB 串行芯片。为了让 Windows 将其识别为 COM 端口，需要安装相应的驱动程序。

下载 CH340 驱动程序 (wch-ic.com)

运行下载的文件 CH341SER.EXE → 点击窗口中的 "INSTALL"→完成。无需重新启动。

您使用 macOS 还是 Linux？
  • macOS：从 macOS Big Sur 开始，驱动程序已包含在系统中，无需任何安装步骤。
  • Linux：已安装内核驱动程序。在 Ubuntu 上，BRLTTY 服务会抢占 CH340，并在几秒钟后使设备文件再次消失--卸载 BRLTTY 会有帮助。您的用户还必须在 dialout 组中。这两个步骤在 GETTING_STARTED.md 中都有详细说明。

步骤 3 - 将加密狗插入电脑

将加密狗直接插入电脑上空闲的 USB 端口。电路板上的小型 LED 灯会立即亮起。

步骤 4 - 固件闪存

点击按钮。然后浏览器会询问授权访问哪个连接设备。

选择对话框中会显示计算机上所有串行设备的列表。选择名称为 "USB-SERIAL CH340" (Windows) 或 "USB Single Serial" 或 "wch.cn" (macOS/Linux) 的条目，并用 "Connect"确认。如果显示多个 CH340 设备，是因为恰好连接了其他带有该芯片的设备，请暂时断开其他设备。

用 "安装"确认安全提示 "擦除设备并安装 PhoneBlock 加密狗？在第一次闪存过程中，整个内存将被擦除，固件将被安装--这需要大约一分钟的时间。在此期间不要断开加密狗。

步骤 5 - 将加密狗插入路由器

当浏览器报告 "安装完成！"时，从电脑上取下加密狗，并将其插入路由器上的空闲 USB 端口（或 WLAN 范围内的任何 5 V USB 电源）。

步骤 6 - 将加密狗添加到无线局域网（WPS）

刚刷新的加密狗还没有任何 Wi-Fi 访问数据，正在等待通过 WPS 配对。LED 指示灯快速闪烁（每秒约闪烁五次），发出 "配对模式激活 "的信号。

现在按下路由器上的 WPS 按钮。在 Fritz！Box 上，这是设备顶部的 "连接/WPS"按钮 - 短按，直到路由器上的 Wi-Fi LED 指示灯开始闪烁。其他路由器则将该按钮标为 "WPS "或 "配对"。几秒钟后，路由器就会将您的 Wi-Fi 访问数据传输给加密狗，加密狗会保存数据并进行连接。

已工作： LED 指示灯现在缓慢闪烁（大约每秒一次）--加密狗已进入 WLAN，已分配 IP，正在等待步骤 7 中的配置。

不起作用请在路由器设置中检查是否激活了 WPS（在 Fritz!Box 上：WLAN → 安全 → WPS）。如果 WPS 窗口太短，加密狗会自动重新启动配对过程，这时只需再次按下 WPS 按钮即可。

第 7 步 - 输入 SIP 访问权限和 PhoneBlock 令牌

为了让加密狗拦截通话，它还需要两样东西：您的 SIP 访问数据（以便它能作为附加电话登录路由器）和 PhoneBlock 令牌（以便它能查询垃圾邮件数据库）。您可以在加密狗的网页界面上输入这两样东西。

在浏览器中打开：

http://answerbot/

如果此链接不起作用（没有 Bonjour 的 Windows 系统、某些网状 WLAN），请在路由器的设备列表中查找名称为 answerbot 的设备，记下其 IP 地址并在浏览器中调用，例如 http://192.168.178.xxx/。

输入网络界面：

  • SIP 访问数据 - 用户名、密码和注册商地址。对于 Fritz！Box，请在 Telephony → Telephony devices → Set up new device → Telephone (with and without answering machine) → LAN/WLAN (IP telephone) 下创建附加连接。在此输入分配的访问数据，并输入 fritz.box 作为注册商。
  • PhoneBlock 标记 - 在PhoneBlock 设置中创建一个 API 标记，并将其复制到相应字段。

保存→LED 指示灯长亮→加密狗准备就绪。现在，垃圾电话将在手机响铃前被悄悄拦截。

有什么问题吗？您使用的是 Firefox/Safari，Linux 有特殊问题，还是另一块板子？详细说明包括所有选项、Linux 漏洞和 ESP-IDF 工具链，请参阅软件仓库：GETTING_STARTED.md。

提交经验报告

在家里的 Fritz！Box 上，加密狗运行稳定，但路由器型号、供应商和连接类型五花八门，每种组合都有自己的特点：不同的 SIP 方言、不同的 NAT 规则、不同的振铃延迟。为了让我能够识别模式并有针对性地解决问题，结构化经验报告的价值不言而喻--无论您是一切顺利还是出现了问题。

尤其有趣的是，这些设置的覆盖面仍然很窄：

  • 使用 Speedport 路由器的 Telekom 连接
  • 通过提供的提供盒连接 Vodafone/Unitymedia 电缆
  • 1&1、o2、Deutsche Glasfaser 和其他供应商
  • 国际连接（瑞士、奥地利、欧洲其他国家）

报告通过 GitHub 问题运行--预先填写的表格以结构化的方式要求提供最重要的关键数据（硬件、路由器、提供商、固件版本、结果）。其他测试人员可直接查看哪些设置已涵盖，哪些未涵盖。
