PhoneBlock 加密狗

一台披着 IP 电话外衣的智能应答机。它只有拇指大小，可插入路由器的 USB 端口，注册为一个额外的 SIP 电话，并在实际电话响起之前拦截商业来电。无论路由器型号如何，即使没有 Fritz 也能使用！

状态：测试版。第一个版本已经可靠运行。我正在寻找更多的测试人员，以便在尽可能多的不同环境中试用加密狗，见下文。

一句话说清想法

加密狗像一个额外的 IP 电话一样登录路由器。每次有电话打进来，它都会以闪电般的速度询问 PhoneBlock 数据库："这是垃圾邮件吗？"如果是，它就会自己接听电话，如果需要，还会播放一段简短的广播，然后再次挂断电话。此时，您的真实电话甚至还没有响起。与云答录机相比，所有呼叫处理都是在您家中本地进行的。

第一个电话打进来

路由器向包括加密狗在内的所有注册电话发出呼叫信号。真实 "电话可配置 3-5 秒的铃声延迟。

2. 闪光灯检查

加密狗会以迅雷不及掩耳之势询问 PhoneBlock 该号码是否为垃圾号码。查询通过加密的 HTTPS 进行，只传输号码的哈希值。电话簿中联系人的来电甚至都不会被检查--它们永远不会离开你的家庭网络。

3. 摆脱它

如果号码是垃圾邮件，加密狗就会接听，播放一段广播，然后挂断。客厅里的电话则保持沉默。如果号码不是已知的垃圾号码，加密狗就什么也不做，电话正常接通。

它为何如此迷人：加密狗只需最低限度的电源和 Wi-Fi 即可使用。无需额外的服务器，无需 Fritz！Box，无需合同。内部的 ESP32 微控制器耗电不到半瓦，您只需将其插入路由器的 USB 端口，然后就可以不用管它了。

比较： 拦截固定电话垃圾邮件的三种方法

所有这三种解决方案都使用由 PhoneBlock 用户维护的相同垃圾邮件数据库，但在工作量、有效性和对连接的要求方面有很大不同。

哪种解决方案适合您？

  • Dongle - 如果您希望获得最佳的垃圾邮件防御和最佳的数据保护。
  • Answer Bot - 您拥有自己的 Fritz！Box（非租用 Box），并接受 PhoneBlock 服务器对您的所有呼叫进行检查，以实现最佳的垃圾邮件防御。
  • Blocklist 电话簿 - 如果您有一个 Fritz！Box（也是一个租用盒），而且您不介意偶尔有垃圾邮件发送者溜进来，但他确实在拦截列表上。

先决条件

  • SIP 注册器 - 如果允许外部 SIP 客户端（"可以注册 VOIP 电话"），可以在路由器中注册，也可以直接与电话提供商注册。
  • 2.4 GHz WLAN 范围内。
  • 5 V USB 电源 - 最好是路由器的 USB 端口，或任何 USB 电源装置。
  • 免费 PhoneBlock 帐户</x1.

提交经验报告

加密狗在我家的 Fritz!Box 上运行稳定，但路由器型号、供应商和连接类型五花八门，每种组合都有自己的特点：不同的 SIP 方言、不同的 NAT 规则、不同的环路延迟选项。为了帮助我识别模式和解决问题，结构化经验报告的价值不言而喻。

尤其有趣的是，这些设置的覆盖面仍然很窄：

  • 使用 Speedport 路由器的 Telekom 连接
  • 通过提供的提供盒连接 Vodafone/Unitymedia 电缆
  • 1&1、o2、Deutsche Glasfaser 和其他供应商
  • 国际连接（瑞士、奥地利、欧洲其他国家）

报告通过 GitHub 问题运行--预先填写的表格以结构化的方式要求提供最重要的关键数据（硬件、路由器、提供商、固件版本、结果）。其他测试人员可直接查看哪些设置已涵盖，哪些未涵盖。

开放体验报告

自己建造

加密狗完全开源。如果您精通 ESP32（或想成为 ESP32 专家），您可以自己制作一个并立即开始使用 - 材料成本约为 8-12 欧元，具体取决于您在哪里订购部件。

硬件

您只需一个带有 ESP-WROOM-32E 模块（如 NodeMCU-32E）的标准 ESP32 DevKit。无需焊接，只需用短 USB 电缆连接到路由器即可。详细的模块和电路板购买说明可在软件仓库中找到。

固件

基于 ESP-IDF 的开源固件。可通过 USB 直接在 Chrome 浏览器或 Edge 浏览器中闪存，也可使用 ESP-IDF 工具链直接闪存。整个初始配置（WLAN、SIP、令牌）可通过一个小型网络前端直接在加密狗上完成。
