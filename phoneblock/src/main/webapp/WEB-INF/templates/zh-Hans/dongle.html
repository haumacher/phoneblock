PhoneBlock 加密狗 New

一台披着 IP 电话外衣的智能应答机。它只有拇指大小，可插入路由器的 USB 端口，注册为一个额外的 SIP 电话，并在实际电话响起之前拦截商业来电。无论路由器型号如何，即使没有 Fritz 也能使用！

状态：测试版第一个版本已经可以稳定运行。我正在寻找更多的测试人员，以便在尽可能多的不同环境中试用加密狗 - 欢迎提交经验报告。

一句话说清想法

加密狗像一个额外的 IP 电话一样登录路由器。每次有电话打进来，它都会以闪电般的速度询问 PhoneBlock 数据库："这是垃圾邮件吗？"如果是，它就会自己接听电话，如果需要，还会播放一段简短的广播，然后再次挂断电话。此时，您的真实电话甚至还没有响起。与云答录机相比，所有呼叫处理都是在您家中本地进行的。

第一个电话打进来

路由器向包括加密狗在内的所有注册电话发出呼叫信号。真实 "电话可配置 3-5 秒的铃声延迟。

2. 闪光灯检查

加密狗会以迅雷不及掩耳之势询问 PhoneBlock 该号码是否为垃圾号码。查询通过加密的 HTTPS 进行，只传输号码的哈希值。电话簿中联系人的来电甚至都不会被检查--它们永远不会离开你的家庭网络。

3. 摆脱它

如果号码是垃圾邮件，加密狗就会接听，播放一段广播，然后挂断。客厅里的电话则保持沉默。如果号码不是已知的垃圾号码，加密狗就什么也不做，电话正常接通。

它为何如此迷人：加密狗只需最低限度的电源和 Wi-Fi 即可使用。无需额外的服务器，无需 Fritz！Box，无需合同。内部的 ESP32 微控制器耗电不到半瓦，您只需将其插入路由器的 USB 端口，然后就可以不用管它了。

这就是它的实用之处：说明书一步一步地指导你进行订购和设置--大约 10 欧元的硬件，只需几分钟的一次性努力。

步骤 2 - 准备加密狗

您需要将 PhoneBlock 软件加载到加密狗上。为此，请将加密狗插入电脑的空闲 USB 端口。

此浏览器无法闪烁加密狗。要准备使用加密狗，您需要在电脑上（非智能手机上）安装 Google Chrome 或 Microsoft Edge 浏览器。用 Chrome 浏览器或 Edge 浏览器打开此页面，然后再试一次。

点击按钮。然后，浏览器会询问允许其寻址的连接设备。选择名称为 "USB-SERIAL CH340" (Windows) 或 "USB Single Serial" 或 "wch.cn" (macOS/Linux) 的条目，然后用 "连接"确认。

选择对话框中将显示计算机上所有串行设备的列表。如果显示了多个 CH340 设备，是因为恰好连接了另一个带有该芯片的设备，请暂时移除其他设备。

电脑无法识别加密狗？驱动程序提示

加密狗使用 CH340 USB 串行芯片。当前的 Windows、macOS 和 Linux 版本通常已包含驱动程序，在这种情况下，你不需要做任何事情。不过，如果你的电脑在设备选择中没有显示加密狗，下面的提示会有所帮助：

  • Windows：从 wch-ic.com 下载 CH340 驱动程序，执行 CH341SER.EXE 并单击 "安装"。无需重新启动。
  • macOS：从 macOS Big Sur 开始，驱动程序已包含在系统中，无需任何安装步骤。
  • Linux：已安装内核驱动程序。在 Ubuntu 上，BRLTTY 服务可能会抢走 CH340，使你无法看到设备文件 - 在这种情况下，卸载 BRLTTY 会有帮助：sudo apt remove --purge brltty。你的用户还必须在 dialout 组中：sudo usermod -aG dialout $USER。这两个步骤在 GETTING_STARTED.md 中都有详细说明。

步骤 3 - 将加密狗插入路由器

当浏览器报告 "安装完成！"时，从电脑上取下加密狗，并将其插入路由器上的空闲 USB 端口（或 WLAN 范围内的任何 5 V USB 电源）。

步骤 4 - 将加密狗添加到无线局域网（WPS）

刚刷新的加密狗还没有任何 Wi-Fi 访问数据，正在等待您将其连接到 Wi-Fi 上。LED 指示灯快速闪烁（每秒约闪烁五次），发出 "配对模式激活 "的信号。

现在按下路由器上的 WPS 按钮。在 Fritz！Box 上，这是设备顶部的 "连接/WPS"按钮 - 短按，直到路由器上的 Wi-Fi LED 指示灯开始闪烁。其他路由器则将该按钮标为 "WPS "或 "配对"。几秒钟内，路由器就会将您的 Wi-Fi 访问数据传输给加密狗，加密狗会保存数据并进行连接。

已工作： LED 指示灯现在缓慢闪烁（大约每秒一次）--加密狗已进入 WLAN，收到地址并等待配置（下一步）。

不起作用请在路由器设置中检查是否激活了 WPS（在 Fritz!Box 上：WLAN → Security → WPS）。如果 WPS 窗口太短，加密狗会自动重新启动配对 - 然后只需再次按下 WPS 按钮即可。

第 5 步 - 注册手机并设置 PhoneBlock 访问权限

为了让加密狗拦截通话，它还需要两样东西：您的 SIP 访问数据（以便它能作为附加电话登录）和 PhoneBlock 令牌（以便它能查询垃圾邮件数据库）。您可以在加密狗的网页界面上输入这两样东西。

在浏览器中打开：

http://answerbot/

已识别加密狗： http://192.168.178.xxx/ - 顶部的按钮可直接打开。

如果此链接不起作用（没有 Bonjour 的 Windows 系统、某些网状 WLAN），请在路由器的设备列表中查找名称为 answerbot 的设备，记下其 IP 地址并在浏览器中调用，例如 http://192.168.178.xxx/。

输入网络界面：

  • SIP 访问数据 - 用户名、密码和注册商地址。对于 Fritz！Box，请在 Telephony → Telephony devices → Set up new device → Telephone (with and without answering machine) → LAN/WLAN (IP telephone) 下创建附加连接。在此输入分配的访问数据，并输入 fritz.box 作为注册商。
  • PhoneBlock 标记 - 在PhoneBlock 设置中创建一个 API 标记，并将其复制到相应字段。

保存→LED 指示灯长亮→加密狗准备就绪。现在，垃圾电话将在您的手机响起之前被悄悄拦截。

有什么问题吗？您使用的是 Firefox/Safari，Linux 有特殊问题，还是另一块板子？详细说明包括所有选项、Linux 漏洞和 ESP-IDF 工具链，请参阅软件仓库：GETTING_STARTED.md。
