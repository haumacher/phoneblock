PhoneBlock 加密狗

一台披着 IP 电话外衣的智能应答机。它只有拇指大小，可插入路由器的 USB 端口，注册为一个额外的 SIP 电话，并在实际电话响起之前拦截商业来电。无论路由器型号如何，即使没有 Fritz 也能使用！

状态：测试版加密狗在我的 Fritz!Box 上运行稳定，但在广泛推荐之前，我希望能在尽可能多的不同类型连接上试用。我正在寻找测试者，并赠送设备作为回报--见下文。

一句话说清想法

加密狗像一个额外的 IP 电话一样登录路由器。每次有电话打进来，它都会以迅雷不及掩耳之势询问 PhoneBlock 数据库："这是垃圾邮件吗？"如果是，它就会自己接听电话，并根据需要播放一段简短的问候语，然后再次挂断电话。此时，您真正的电话还没有响起。

第一个电话打进来

路由器向包括加密狗在内的所有注册电话发出呼叫信号。真实 "电话可配置 3-5 秒的铃声延迟。

2. 闪光灯检查

加密狗会以迅雷不及掩耳之势询问 PhoneBlock 该号码是否为垃圾号码。查询通过加密的 HTTPS 进行，只传输号码的哈希值。电话簿中联系人的来电甚至都不会被检查--它们永远不会离开你的家庭网络。

3. 摆脱它

如果号码是垃圾邮件，加密狗就会接听，并根据需要播放广播，然后挂断电话。客厅里的电话则保持静音。如果号码不是垃圾号码，加密狗什么也不做，电话照常接通。

为何如此迷人：加密狗只需最低限度的电源和 Wi-Fi。不需要额外的服务器，不需要 Fritz！Box 的特殊功能，也不需要合同。内部的 ESP32 微控制器耗电不到半瓦，您只需将其插入路由器的 USB 端口，然后就可以不用管它了。

哪种解决方案适合您？

  • Answer Bot - 如果您想积极雇用垃圾邮件发送者，并且您的 Fritz!Box 允许从互联网登录 SIP。
  • 加密狗 - 如果您没有Fritz！Box 或不想运行永久服务器，但仍希望在收到垃圾邮件时手机能保持可靠的静音。
  • Blocklist 电话簿 - 如果您有一个 Fritz!Box，那么一个简单的标记或手动路由器呼叫拦截对您来说就足够了，无论如何，您都不会介意真正在拦截名单上的垃圾邮件发送者漏网。

先决条件

  • 带有 SIP 注册器的路由器或电话系统 - Fritz!Box、Speedport、Provider-Box、Asterisk、FreePBX、3CX 和类似系统。
  • 为 "真实 "电话配置铃声延迟（通常为 3-5 秒），以便加密狗可以提前拦截通话。
  • 2.4 GHz WLAN 范围内。
  • 5 V USB 电源 - 最好直接安装在路由器上，否则可安装在任何 USB 电源装置上。
  • 免费 PhoneBlock 帐户，带 API 访问令牌。

申请成为测试员

自己建造

加密狗完全开源。如果您精通 ESP32（或想成为 ESP32 专家），您可以自己制作一个 - 材料成本约为 8-12 欧元，具体取决于您在哪里订购部件。

硬件

您只需一个带有 ESP-WROOM-32E 模块（如 NodeMCU-32E）的标准 ESP32 DevKit。无需焊接，只需用短 USB 电缆连接到路由器即可。详细的模块和电路板购买说明可在软件仓库中找到。

固件

基于 ESP-IDF 的开源固件。可使用 ESP-IDF 工具链通过 USB 或现成的二进制文件进行更新。完整的初始配置（WLAN、SIP、令牌）可通过一个小型网络前端直接在加密狗上运行。

源代码和说明

硬件选择、闪存说明、配置和故障排除--全部都在 GitHub 存储库中。进一步的开发工作也在这里进行；欢迎提出拉取请求和问题。
