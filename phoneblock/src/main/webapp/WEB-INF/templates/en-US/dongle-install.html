Flashing the dongle in the browser

One ESP32 board, one USB cable, one click - the latest PhoneBlock firmware is transferred directly from this browser to your dongle.

Your browser does not support web serial Web flashing only works in desktop Chrome, Edge or Opera. On Firefox, Safari, Android and iOS, you can flash the dongle with the ESP-IDF toolchain via USB instead.

1. prepare hardware

  • ESP32-DevKit with ESP-WROOM-32E (e.g. NodeMCU-32E) - see Hardware notes.
  • USB-data cable (not just a charging cable) between computer and dongle.
  • On Windows you may need to install the USB serial driver - usually CP210x (Silicon Labs) or CH340/CH341 (WCH).
  • On Linux your user must be in the dialout group (sudo usermod -aG dialout $USER, then log out and back in once).

2. flashing

Click on the button, select your dongle in the browser dialog and wait until the process is complete. During an initial flash, the flash memory is deleted - any existing configuration will be lost.

3. after flashing

  1. The dongle restarts and opens the WLAN PhoneBlock-Dongle-XXXX.
  2. Switch to this WLAN with your smartphone or notebook, the browser opens the setup page automatically (captive portal).
  3. Select home WLAN, enter SIP access data and PhoneBlock token - details in the Dongle instructions.

Source code of the firmware: phoneblock-dongle on GitHub.
