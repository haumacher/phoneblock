PhoneBlock Dongle

A thumb-sized USB stick that snatches away advertising calls before your phone even rings - and which is not tied to the Fritz!Box, but works on any modern router or telephone system with SIP.

Status: Beta. The dongle works, but before I recommend it widely, I would like to have tried it out on as many different connection types as possible. I am looking for testers and am giving away devices in return - see below.

The idea in one sentence

The dongle logs on to your router like an additional IP phone. Every time a call comes in, it asks the PhoneBlock database at lightning speed: "Is this spam?" If so, it answers the call itself, plays a short announcement if desired and hangs up again. Your real phone hasn't even rung at that moment.

1st call comes in

The router signals the call to all registered telephones - including the dongle. The "real" telephones are configured with a ring delay of 3-5 seconds.

2. flash check

The dongle asks PhoneBlock within a few hundred milliseconds whether the number is known to be spam. The query runs via encrypted HTTPS and only transmits a hash of the number.

3. get rid of it

If the number is spam, the dongle picks up, plays an announcement if desired and hangs up. The telephone in the living room remains silent. If it is harmless, the dongle simply does nothing - the call goes through normally.

Why this is so charming: The dongle needs nothing but power and Wi-Fi. No additional server, no special Fritz!Box function, no contract. The ESP32 microcontroller inside draws less than half a watt - you can simply plug it into the USB port of your router and forget about it.

Which solution suits you?

  • Blocklist phonebook - if you have a Fritz!Box and a simple marking or the manual router call blocking is sufficient for you.
  • Dongle - if you don't have a Fritz!Box or don't want to operate a permanent server, but still want the phone to be reliably silent in the event of spam.
  • Answer Bot - if you want to actively employ spammers and your router allows a SIP login from the Internet.

Prerequisites

  • Router or telephone system with SIP registrar - Fritz!Box, Speedport, Provider-Box, Asterisk, FreePBX, 3CX and similar.
  • Configurable ring delay for the "real" telephones (typically 3-5 seconds) so that the dongle can intercept the call beforehand.
  • 2.4 GHz WLAN within range.
  • 5 V USB power supply - ideally directly on the router, otherwise on any USB power supply unit.
  • Free PhoneBlock account with API access token.

Build it yourself

The dongle is completely open source. If you are ESP32-savvy (or want to become one), you can build one yourself - the material costs are around 8-12 €, depending on where you order the parts.

Hardware

A commercially available ESP32 DevKit with ESP-WROOM-32E module (e.g. NodeMCU-32E) is sufficient. No soldering required - simply connect to the router with a short USB cable. Detailed module and board purchase instructions can be found in the repo.

Firmware

Open source firmware based on ESP-IDF. Flashing via USB with the ESP-IDF toolchain or later via a ready-made binary. The complete initial configuration (WLAN, SIP, token) runs via a small web frontend directly on the dongle.

Source code & instructions

Hardware selection, flash instructions, configuration and troubleshooting - all in the GitHub repository. Further development is also taking place there; pull requests and issues are welcome.
