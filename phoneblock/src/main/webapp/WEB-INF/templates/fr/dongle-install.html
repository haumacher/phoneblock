Construire son propre dongle - étape par étape

Suis les instructions dans l'ordre. Une fois terminé, le dongle se branche sur le port USB de ton routeur et bloque silencieusement les appels de spam en arrière-plan.

Status : Beta. Ces instructions sont conçues pour un seul type de matériel testé - achetez exactement la carte de l'étape 1, et toutes les autres étapes seront adaptées. D'autres cartes fonctionnent souvent aussi, mais nécessitent d'autres pilotes et votre propre travail de détective (voir le lien ci-dessous).

Ce navigateur ne peut pas flasher le dongle. Le flashage web ne fonctionne que dans Google Chrome ou Microsoft Edge sur un ordinateur (pas sur un smartphone). Ouvre cette page avec Chrome ou Edge et réessaie.

Étape 1 - Commander le dongle

Commande cette carte ESP32. Elle est suffisamment compacte pour être insérée directement comme clé USB dans le routeur, elle possède la puce série USB appropriée (CH340), et c'est avec elle que le dongle PhoneBlock a été développé et testé.

Livraison depuis la Chine - environ 3-4 semaines de délai de livraison. Prix actuel : environ 9 €, frais de port inclus.

Commander sur AliExpress

Tu veux prendre une autre carte? D'autres DevKits ESP32 fonctionnent techniquement de la même manière, mais peuvent avoir besoin d'un autre pilote série USB (par ex. CP2102 au lieu de CH340) et d'un câble d'extension USB, si la forme de construction ne convient pas directement au routeur. Détails et recommandations : HARDWARE.md dans le repo.

Étape 2 - Installer le pilote USB (Windows uniquement)

Le dongle utilise la puce sérielle USB CH340. Pour que Windows le reconnaisse comme port COM, tu as besoin une seule fois du pilote approprié.

Télécharger le pilote du CH340 (wch-ic.com)

Exécuter le fichier téléchargé CH341SER.EXE → cliquer dans la fenêtre sur "INSTALL" → terminé. Pas de redémarrage nécessaire.

Tu utilises macOS ou Linux ?
  • macOS : à partir de macOS Big Sur, le pilote est inclus dans le système - aucune étape d'installation n'est nécessaire.
  • Linux : le pilote du noyau est intégré. Sur Ubuntu, le service BRLTTY peut s'emparer du CH340 et faire disparaître le fichier du périphérique après quelques secondes - la désinstallation de BRLTTY peut alors aider. Ton utilisateur doit également être dans le groupe dialout. Ces deux étapes sont détaillées dans GETTING_STARTED.md.

Étape 3 - Insérer la clé électronique dans l'ordinateur

Branche le dongle directement dans un port USB libre de ton ordinateur. Une petite LED sur la carte devrait s'allumer ou clignoter immédiatement.

Étape 4 - Flasher le firmware

Cliquer sur le bouton. Le navigateur demande ensuite à quel appareil connecté il peut s'adresser.

La boîte de dialogue de sélection affiche une liste de tous les appareils sériels de l'ordinateur. Sélectionne l'entrée avec "USB-SERIAL CH340" (Windows) ou "USB Single Serial" ou "wch.cn" (macOS/Linux) dans son nom et confirme avec "Connecter". Si plusieurs appareils CH340 s'affichent parce qu'il se trouve qu'un autre appareil avec cette puce est connecté, retire les autres pour le moment.

Confirme la question de sécurité "Erase device and install PhoneBlock Dongle ?" avec "Install". Lors du premier flash, toute la mémoire est effacée et le firmware est installé - cela dure environ une minute. Ne déconnecte pas le dongle pendant ce temps.

Étape 5 - Replacer la clé électronique dans le routeur

Lorsque le navigateur indique "Installation complete !", retire le dongle de l'ordinateur et branche-le sur un port USB libre de ton routeur (ou sur n'importe quel bloc d'alimentation USB 5 V à portée de WLAN).

Étape 6 - Ajouter la clé électronique au réseau sans fil (WPS)

Le dongle fraîchement flashé n'a pas encore de données d'accès WLAN et attend l'appairage par WPS. La LED clignote rapidement (environ cinq fois par seconde) et signale "mode d'appairage actif".

Appuie maintenant sur le bouton WPS du routeur. Sur la Fritz!Box, il s'agit du bouton "Connect / WPS" sur le dessus de l'appareil - appuie brièvement jusqu'à ce que le voyant WLAN du routeur commence à clignoter. D'autres routeurs indiquent "WPS" ou "Pair" sur le bouton. En quelques secondes, le routeur transmettra tes données d'accès WLAN au dongle, qui les enregistrera et se connectera.

Ça a marché: La LED clignote maintenant lentement (environ une fois par seconde) - le dongle est sur le WLAN, a reçu une IP et attend la configuration à l'étape 7.

Ça n'a pas marché? Vérifie dans les paramètres du routeur si le WPS est vraiment activé (sur la Fritz!Box : WLAN → Sécurité → WPS). Si la fenêtre WPS a été trop courte, le dongle redémarre automatiquement l'appairage - il suffit alors d'appuyer à nouveau sur le bouton WPS.

Étape 7 - Inscrire l'accès SIP et le token PhoneBlock

Pour que le dongle puisse intercepter des appels, il lui manque encore deux choses : tes données d'accès SIP (pour qu'il puisse se connecter au routeur en tant que téléphone supplémentaire) et un token PhoneBlock (pour qu'il puisse interroger la base de données des spams). Tu inscris ces deux éléments dans l'interface web du dongle.

Ouvre dans le navigateur :

http://answerbot.local/

Si ce lien ne fonctionne pas (Windows sans Bonjour, certains réseaux locaux sans fil maillés), consulte la liste des périphériques de ton routeur pour trouver un périphérique nommé answerbot, note son adresse IP et accède à celle-ci dans ton navigateur - par exemple http://192.168.178.42/.

Inscris-toi dans l'interface web :

  • Données d'accès SIP - nom d'utilisateur, mot de passe et adresse du registraire. Dans le cas d'une Fritz!Box, tu crées pour cela un raccordement supplémentaire sous Téléphonie → Appareils de téléphonie → Configurer un nouvel appareil → Téléphone (avec et sans répondeur) → LAN/WLAN (téléphone IP). Les données d'accès qui y sont attribuées entrent ici, tu inscris fritz.box comme registraire.
  • Token PhoneBlock - crée un token API dans tes paramètres PhoneBlock et copie-le dans le champ correspondant.

Enregistrer → la LED est maintenant allumée en permanence → le dongle est prêt à fonctionner. Les appels de spam sont désormais interceptés silencieusement avant que ton téléphone ne sonne.

Quelque chose ne fonctionne pas, tu utilises Firefox/Safari, Linux avec des particularités ou un autre board ? Les instructions détaillées avec toutes les options, les pièges de Linux et la toolchain ESP-IDF se trouvent dans le repo : GETTING_STARTED.md.
