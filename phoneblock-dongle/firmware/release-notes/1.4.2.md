# PhoneBlock Dongle 1.4.2

*2026-07-10*

## Fixed

- **No more watchdog reboots when the connection to the provider goes
  dead.** If the network path to the SIP provider broke while the dongle
  was in the middle of sending a registration — the link silently dropped,
  a mobile-hotspot uplink stalled, the provider's edge stopped responding —
  the send could hang indefinitely with no timeout. The registration task
  then stopped feeding the watchdog and the dongle rebooted itself after a
  minute. The dongle now bounds every send to the provider: a stalled
  connection is given up on and reopened within a few seconds, so it
  recovers by reconnecting instead of restarting.
