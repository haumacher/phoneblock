# PhoneBlock Dongle 1.5.1

*2026-07-19*

## Security

- **A crafted call-teardown message from the network can no longer crash
  the dongle.** When answering certain in-dialog SIP messages (such as a
  BYE ending a call), the dongle echoed parts of the incoming message back
  into its reply. An unusually large such message could write past the end
  of the reply buffer and corrupt neighbouring memory, which surfaced later
  as a crash — this is the fault behind the reboot some 1.5.0 dongles saw.
  Normal Fritz!Box and provider traffic never sends messages that big, so
  the effect was rare, but a deliberately oversized one reaching the dongle
  could trigger it. The reply builder is now overflow-proof by
  construction: it stops cleanly at the buffer's edge instead of writing
  beyond it, so oversized input is simply truncated and can no longer
  corrupt memory or crash the device. Ordinary replies are unchanged.

## Changed

- **The dashboard loads faster and leaves more room for future updates.**
  The dongle's web dashboard is now stored and delivered compressed. The
  page shrinks from about 204 KB to 53 KB on the wire — browsers unpack it
  transparently, so it looks and behaves exactly as before — and the
  smaller firmware image frees up flash headroom in the update slot (from
  under 5 % spare to over 13 %), giving later firmware versions room to
  grow.

## Fixed

- **Status e-mails now carry the correct send time.** The dongle's status
  mails went out without a Date header, so mail clients showed the time the
  message arrived (or none at all) rather than when the dongle actually
  sent it. The mail now stamps its own send time, taken from the dongle's
  network-synced clock. If the clock has not synced yet, the header is left
  out so the receiving mail server fills in an accurate time instead of a
  wrong one.
