# PhoneBlock Dongle 1.4.1

*2026-07-06*

## Fixed

- **No more spurious "STUN gave no mapping" warning on a Fritz!Box.** The
  1.4.0 STUN feature is only meant for registering straight at a provider
  behind a home router; on a Fritz!Box the dongle registers on the local
  network and its media reaches the box directly, so there is no NAT to
  look up. The dongle nevertheless attempted a STUN query on every
  incoming call and, finding no STUN server configured, logged an alarming
  "STUN gave no mapping — advertising local RTP endpoint …" warning to the
  diagnostics panel — even though the local endpoint it announced was
  exactly correct. The dongle now skips STUN entirely when no server is
  configured (the Fritz!Box default) and announces the local endpoint
  quietly, so the panel no longer shows a fault where there is none.
