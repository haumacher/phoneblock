# PhoneBlock Dongle 1.3.5

*2026-06-18*

## Fixed

- **No more reboots while using the web interface.** The task-watchdog
  was effectively stuck at a 5 s timeout, so ordinary web actions that
  briefly block the dongle's server could exceed it and reboot the
  device: testing the token (which contacts the PhoneBlock server over
  TLS) or loading the status page over a slow Wi-Fi link. The watchdog
  timeout is now 60 s, which comfortably covers these operations while
  still catching a genuine hang within a minute. (The 120 s value
  announced for 1.3.4 was never actually applied — it is above the
  platform limit and was silently ignored.)
