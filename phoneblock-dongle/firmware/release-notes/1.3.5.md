# PhoneBlock Dongle 1.3.5

*2026-06-18*

## Fixed

- **No more reboots while using the web interface.** The task-watchdog
  timeout raise to 120 s announced for 1.3.4 never actually made it into
  that build — a stale build configuration kept the old 5 s value. As a
  result, ordinary web actions that briefly block the server could exceed
  the watchdog window and reboot the dongle: testing the token (which
  contacts the PhoneBlock server over TLS) or loading the status page over
  a slow Wi-Fi link. The 120 s timeout now ships as intended.
