# PhoneBlock 3.1.2

*2026-06-14*

## Key Features

- **Personal wildcard blocks (#377).** Manage personal wildcard (range) blocks from the web UI, grouped under their respective sections. The wildcard prefix floor is lowered to 3 digits to allow country- or city-wide blocks, and a wildcard block subsumes existing exact blocks under the same prefix.

## API

- Personal wildcards sync via the API (`PersonalizedNumber.wildcard`)
- Wildcard catches produce a weighted call report
- `/api/check-prefix` `RangeMatch.votes` is derived from decoded evidence instead of the now-removed `VOTES` column
- `PhoneInfo.archived` restored as a deprecated constant `false` for backward compatibility

## CardDAV

- Personal wildcard blocks are emitted into the address book

## Web App

- Block-spam decision is now recompute-based (counts current members, spread × mass)

## Database

Schema version 42. Migration 41 adds the `WILDCARD` column to `PERSONALIZATION`; migration 42 drops the now-dead `VOTES` column from the aggregation tables and adds the recompute columns (`MEMBERS`, `TENS`).
