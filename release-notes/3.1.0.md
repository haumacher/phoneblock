# PhoneBlock 3.1.0

*2026-06-13*

## Key Features

- **Decay-aware spam scoring (epic #300).** Votes, call reports and searches now feed time-decaying exponential moving averages ("Heat"), so stale reports fade and currently-active spammers rank higher. The raw vote counters are replaced end-to-end by decay-aware evidence.
- **Heat and spam confidence exposed.** A `spamConfidence` value (binomial-proportion confidence) and `heat` (a reports-per-day rate) are surfaced in the API and on the number page, and hidden for legitimate and whitelisted numbers.
- **Heat-ranked, region-aware blocklist.** Space-constrained clients receive a blocklist ranked by Heat and reporting region (#336, #340). Heat-based archiving replaces the old low-vote archiving (#335).
- **Snapshot-driven blocklist publication.** The published blocklist lives in a dedicated `BLOCKLIST` table with bucket-based, snapshot-versioned incremental sync.

## API

- API version bumped to v1.7.0
- `votes` / `votesWildcard` are now decay-aware; `heat` and `spamConfidence` added (#334)
- Wildcard ranges report decoded net evidence; the `/check` wildcard fallback now populates `spamConfidence` and `heat`

## Web App

- Stats page: new calls / votes / searches-per-day activity chart
- Number page: shows spam confidence, heat and the number of intercepted calls; clamps negative "searches today"
- Blacklist entries link to the number-info page

## CardDAV

- Address-book lists ranked by reporting region
- Publishes a heat class instead of raw timestamps, decoding net evidence

## Answer Bot

- Docker image honors `PUID`/`PGID` overrides (#345)

## Database

- Schema version 40. Migrations 27–40 introduce and backfill the decay-aware EMA columns, assign `REVISION.ID` in code instead of via an H2 IDENTITY, batch the nightly history snapshot (30-day retention), index `NUMBERS_HISTORY`, and drop the legacy `RATINGS`, `RATINGHISTORY`, `SEARCHES`, `SPAMREPORTS`, `BLOCKLIST`, `EXCLUDES`, `OLDREPORTS`, and `SEARCHCLUSTER`/`SEARCHHISTORY` tables.
- Added an H2 store health monitor to diagnose database file growth.
- Schema migration now aborts on the first failed statement.
